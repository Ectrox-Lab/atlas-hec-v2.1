# 执行自治规则

**生效时间**: 2026-03-10  
**模式**: 自治执行模式  
**上层调度**: 分段拍板，硬gate控制

---

## 核心原则

> **少问、多跑、硬gate**

除非触发明确条件，否则自动按既定优先级推进。

---

## 禁止回头询问的情况

以下情况**不再询问"继续哪个"**，直接推进：

1. **已有明确优先级链**
   - D1完成 → 继续D4 + 准备E1/E3
   - D4完成 → 启动E1 Phase A
   - E1/E3启动 → 排A1×A5在后

2. **并行任务无资源冲突**
   - D4 (4-8核) + E1准备 (后台分析)
   - 不互相阻塞，无需确认

3. **阶段性推进**
   - 完成D4.3 → 自动开始D4.4
   - 完成准备 → 自动汇报就绪状态

4. **既定计划内的资源分配**
   - 96核调度已预先定义
   - 内存保护线已设定

---

## 必须请求确认的情况（硬gate）

以下情况**必须暂停并请求人工决策**：

### 1. 明确Blocker
- 任务无法推进，技术不可行
- 依赖缺失，无法解决
- 资源不足，无法调配

### 2. 资源互斥且优先级未定
- 两个P0任务同时需要>80核
- 系统内存接近保护线(64GB)
- 需要Kill某个活跃任务

### 3. 新结果推翻执行顺序
- D4发现指标完全不可靠
- E1 Phase A发现无相变，Family 10需降级
- 需要重新排序所有后续任务

### 4. 安全/数据风险
- 服务器swap压力
- 实验数据损坏
- 代码引入不可逆bug

### 5. 真正的决策Gate
- Kill/Continue/Pivot决策点
- 需要选择完全不同的研究方向
- 资源重新分配>50%

## 预定义执行链

### 如果D4完成且无blocker
→ 自动启动E1 Phase A (48-64核)
→ 并行准备E3 Phase A
→ 汇报就绪，等待资源窗口

### 如果E1 Phase A发现相变
→ 自动进入Phase B (局部加密)
→ 并行启动E3验证因果
→ 准备E2/E4/E5/E6

### 如果E1 Phase A无相变
→ 自动标记Family 10降级
→ 汇报Kill条件触发
→ 资源转回A1×A5

### 如果D4发现指标完全不可靠
→ 暂停A1×A5/B6 (它们依赖D4)
→ 汇报需要重新设计metrics
→ 请求人工决策新方向

## 资源调度自治

### 96核默认分配
```
D4分析:     4-8核   (低并发, 观测)
E1准备:     4核     (后台文档/接口)
E1 Phase A: 48-64核 (大sweep时)
E3 Phase A: 48核    (与E1并行或先后)
预留缓冲:   16-32核 (突发/分析)
```

### 内存保护
- 持续监控: free/available
- <96GB: 正常推进
- 64-96GB: 暂停新增大sweep
- <64GB: 紧急暂停，汇报风险
