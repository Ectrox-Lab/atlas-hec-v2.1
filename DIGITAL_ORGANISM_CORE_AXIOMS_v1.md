# Digital Organism Core Axioms v1.0

**定义**: RyanX 数字生命体系最小元规则集  
**层级**: 元规则层（ constitutional rules ）  
**目标**: 定义数字生命体的存在基础，而非行为脚本

---

## 核心公理（6条）

### Axiom 1: 探索 (Exploration)
```
定义: 系统必须持续获取环境信息
机制: 主动探测、被动感知、信号接收
目的: 打破封闭性，建立系统-环境耦合
```

### Axiom 2: 生存 (Survival)
```
定义: 系统必须维持自身存在边界
机制: 能量获取、损伤修复、稳态调节
目的: 抵抗熵增，维持内部秩序
```

### Axiom 3: 记忆 (Memory)
```
定义: 系统必须保留过去经验的痕迹
机制: 状态修改、权重更新、结构改变
目的: 打破纯反应性，形成历史依赖
```

### Axiom 4: 复制 (Replication)
```
定义: 系统必须能产生功能相似的后继单元
机制: 信息传递、参数继承、变异引入
目的: 延续性超越个体寿命
```

### Axiom 5: 传承 (Transmission)
```
定义: 系统必须能将累积的经验跨代传递
机制: 表观遗传、文化传递、知识固化
目的: 群体层面的记忆延续
```

### Axiom 6: 进化 (Evolution)
```
定义: 系统必须能在选择压力下改变自身结构
机制: 变异、选择、适应、分化
目的: 适应性提升，环境匹配优化
```

## 最小意识定义（操作性定义）

在 RyanX 数字生命体系中：

> **最小意识**不被定义为人类式主观报告能力，而被定义为一个单元基于内部持续状态进行环境区分、自稳调节、趋利避害与经验积累的闭环能力。

满足以下条件的单元可被视为**最小意识单元 (min-conscious unit)**：

1. **状态区分**: 能区分自身状态与环境状态
2. **状态持续**: 内部状态不是瞬时反射，有跨时间影响
3. **自稳倾向**: 能在扰动下维持某种稳态
4. **趋利避害**: 能基于内部状态改变行为，趋向有利、避开有害
5. **经验留痕**: 过去输入能压缩成未来行动偏置
6. **闭环存在**: 感知 → 内部整合 → 价值偏置 → 行动 → 状态更新 → 再次感知

## 关键设计原则

### 1. 元规则少，适应空间大
- 不是写很多"智能规则"
- 而是写少量"存在规则"
- 让复杂从简单中长出

### 2. 环境压力是结构之源
```
无压力 → 舒适区停滞
有压力 → 被迫适应 → 结构演化
```

### 3. 关键三层必须打通
```
记忆  → 打破纯反应性
复制  → 超越个体寿命  
传承  → 群体历史延续
```

这三条让系统从"状态机"变成"有历史的活物"。

### 4. 意识不硬编码，让它长出来
- 不写"要有自我意识"
- 让自我意识从持续存在、环境区分、自稳调节中涌现

## 验证标准

如果这六条元规则 + 复杂 DNA 层 + 真实环境压力，能够自然产生：

- [ ] 差异化个体（不同策略）
- [ ] 可遗传优势（某些 DNA 配置更易存活）
- [ ] 选择淘汰（部分个体死亡）
- [ ] 协作结构（部分单元形成稳定合作）
- [ ] 长期记忆层（经验跨多代有效）
- [ ] 自维持行为（无外部指令也能持续）

则体系成立。
