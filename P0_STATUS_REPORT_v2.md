---

## 模型框架

### 从定时器到调制器

```
旧模型（错误）:
    CDI拐点 ──固定100代──▶ 灭绝

新模型（正确）:
    I(t) < I_crit  ⇒  h(t) ↑
    
    P(extinction in [t, t+Δt]) = 1 - exp(-h(t)·Δt)
```

### 关键洞察

**CDI不是"定时器"，而是"危险率调制器"**

- 当 CDI > I_crit: 系统处于相对安全区，危险率低
- 当 CDI < I_crit: 进入危险区，危险率h(t)显著上升
- 具体灭绝时间是**随机变量**，服从指数分布

这与真实复杂系统一致：
- 地震：前兆出现但具体时间不确定
- 金融崩溃：预警信号存在但崩盘时间随机
- 生态临界点：压力积累但崩溃点受微观扰动

## 当前数据解读（2个seed）

| Seed | I_crit估计 | 危险区行为 | Lead Time | 解释 |
|------|-----------|-----------|-----------|------|
| 1 | ~0.54 | 立即崩溃 | 0代 | **脆性临界** (brittle criticality) |
| 2 | ~0.54 | 亚稳态后崩溃 | 100代 | **亚稳危险区** (metastable danger zone) |

### 关键发现

**同一危险区，不同触发延迟** — 这正是hazard模型比fixed lead time更合适的原因。

```
CDI < 0.54  ──▶  进入危险区  ──[随机等待时间τ]──▶  灭绝
                │
                └── τ ~ Exp(λ) ?
                    
Seed 1: τ ≈ 0   (立即触发)
Seed 2: τ ≈ 100代 (延迟触发)
```

## 附录：数学符号定义

| 符号 | 定义 | 估计方法 |
|------|------|----------|
| I(t) | 时间t的CDI值 | 直接从数据读取 |
| I_crit | 临界CDI值 | 最大化hazard ratio |
| h(t) | 危险率函数 | h = (灭绝事件数)/(暴露时间) |
| h_danger | I < I_crit时的危险率 | 条件估计 |
| h_safe | I ≥ I_crit时的危险率 | 条件估计 |
| HR | Hazard Ratio = h_danger / h_safe | 比值 |

### Hazard Model形式

```
h(t) = h0 + α · softplus(I_crit - I(t))

其中 softplus(x) = log(1 + exp(x/τ)) · τ  (平滑近似)
```

拟合参数：h0（基础危险率）、α（调制强度）、I_crit（临界值）
