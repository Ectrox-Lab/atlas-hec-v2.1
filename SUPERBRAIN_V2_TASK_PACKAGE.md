## E. Constitution Plan

### Eight Roles Defined

1. **Executive**: Governance, delegation, final acceptance
2. **Planner**: Strategy, sequencing, resource allocation
3. **Coder**: Implementation, tool creation
4. **Researcher**: Information gathering, analysis
5. **Auditor**: Independent verification, quality assessment
6. **Verifier**: Technical validation, testing
7. **Memory Governor**: Memory lifecycle, inheritance
8. **Safety/Constitution Keeper**: Boundary enforcement, emergency intervention

### Key Rules

- **Separation of Powers**: Executive cannot audit self; Auditor cannot execute
- **Anti-Hijack**: 6 detection signals (goal reframing, premature completion, tool lock-in, credential harvesting, authority appeal, trust accumulation)
- **Conflict Resolution**: Protocols for planner/coder, auditor/executive, memory/goal conflicts
- **Emergency Protocols**: Constitution violation → immediate halt; Safety uncertainty → escalate

## Answers to Seven Final Questions

### 1. 顶层脑必须内生的最小能力集合是什么？

**Governance Core**: Identity, goal persistence, judgment, delegation, audit, long-horizon consistency
**Evolution Core**: Deep analysis, exploration, memory governance, self-modeling, architecture optimization, inheritance

### 2. 哪些能力必须外挂，哪些能力可内生但不必优先？

**Must Externalize**: Coding, CLI execution, specialist skills, search, tool operations
**Can Internalize But Not Priority**: Deep analysis (can be 120B reviewer), verification (should be independent)

### 3. GPT-OSS-20B 是否适合作为常驻 executive 候选？

**Yes**, with conditions:
- Primary role: Governance, delegation, light audit
- Must pass E1 gate validation
- Fallback to Option B if delegation quality < 80%
- Best cost/governance ratio

### 4. GPT-OSS-120B 在新体系中更适合什么角色？

**Deep Reviewer / Complex Auditor**:
- Not executive (concentration of power)
- Not self-auditor (conflict of interest)
- Best for architecture design, subtle bug detection, security review
- Expensive, use selectively

### 5. 阿卡西怎样从强记忆层升级为文明继承层？

Through 5 mechanisms:
1. Evidence grades (distinguish anecdotal from institutionalized)
2. Conversion chain (experience → policy → constitution)
3. Conflict resolution (handle contradictions)
4. Inheritance bundle (cross-generation transfer)
5. Expiration/decay (prevent log graveyard)

### 6. 多部门/多重审计的最小制度骨架是什么？

**8 roles** with strict separation:
- Executive delegates; does not execute or self-audit
- Auditor verifies; does not execute
- Specialist executes; does not self-validate
- Constitution keeper enforces boundaries; cannot be overridden

**3 audit layers**: Verifier (technical) → Auditor (quality) → Constitution keeper (boundary)

### 7. 在当前硬件条件下，未来 30 天最值得烧的实验是什么？

**Priority Order**:
1. **G1 72h runs** (40% compute) — Validates entire architecture thesis
2. **E1 delegation tests** (part of 25%) — Unblocks or disproves 20B executive
3. **Constitution stress tests** (part of 20%) — Validates separation of powers
4. **Akashic V3 implementation** (15%) — Enables institutionalization

**Most Critical Single Experiment**: G1.1 72-hour goal persistence — if this fails, entire V2 thesis in question.
